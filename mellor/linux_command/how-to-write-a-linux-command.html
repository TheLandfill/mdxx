Intro

I'm going to go through a step by step example of how I would generally make a Linux command. For this example, I want the command hello to run the script hello-world.sh, which will print out "Hello World!". For consistency, I will assume hello-world.sh is in the directory ~/dev/example/.

What Exactly is a Linux Command?

If you have any experience with Linux or bash, you'll already know a few standard Linux commands, such as cd (change current directory), mv (move or rename files and folders), and cp (copy files). If you're a little more experienced, you might know about some more complex commands, such as sudo (gives a command temporary admin privileges) or python (runs the python interpretor). Because you can't do much in the terminal without these commands and they come on most terminals, people often assume that they're built in. While a few of them are (cd usually is because it's directly related to the terminal), most are not. They're just programs or scripts in a specific folder. In fact, you can use either the which or type -a to see exactly what folder a command is in. For example: 

joseph@tutorials:~/dev$ which cd
joseph@tutorials:~/dev$ type -a cd
cd is a function
cd ()
{
	builtin cd "$@" && chpwd
}
cd is a shell builtin
joseph@tutorials:~/dev$ which mv
/bin/mv
joseph@tutorials:~/dev$ which cp
/bin/cp
joseph@tutorials:~/dev$ which sudo
/usr/bin/sudo
joseph@tutorials:~/dev$ which python
/usr/bin/python

On my computer, I had to write a function that modified the behavior of cd slightly so that my prompt only displays the current and parent directory, but it doesn't matter for the tutorial. On most of your computers, cd will just be a shell builtin. Since the only way to write a shell builtin is to write or modify a shell, we aren't going to deal with that here.
